## IAM Policy Fixes Applied During This Chunk

Three issues hit during apply — all resolved by updating
`sample-api-terraform-executor-policy`:

| Error | Fix |
|---|---|
| `kms:TagResource` denied | Added KMS statement |
| `logs:CreateLogGroup` denied | Added CloudWatchLogs statement |
| `kms:DeleteKey` invalid action | Removed — `kms:ScheduleKeyDeletion` covers deletion |
| S3 redundant ARN warning | Split into `S3TerraformStateBucket` and `S3TerraformStateObjects` statements |
| OIDC provider ARN mismatch | Added `oidc.eks.us-east-1.amazonaws.com/*` to OIDC statement |
| Fargate profile names didn't match `sample-api-*` | Added `sample-api-` prefix to all profile names |
| `iam:CreateServiceLinkedRole` denied | Fixed path from `fargate.amazonaws.com` to `eks-fargate.amazonaws.com` |
| kubectl credentials error | Added `access_entries` block granting executor role cluster admin access |
| Policy exceeded 6144 char limit | Consolidated verbose action lists using service wildcards |

## access_entries Block Added

Required to allow the Terraform executor role to run kubectl commands:

```hcl
access_entries = {
  terraform_executor = {
    principal_arn = "arn:aws:iam::065571033838:role/sample-api-terraform-executor-role"
    policy_associations = {
      admin = {
        policy_arn   = "arn:aws:eks::aws:cluster-access-policy/AmazonEKSClusterAdminPolicy"
        access_scope = {
          type = "cluster"
        }
      }
    }
  }
}
```
